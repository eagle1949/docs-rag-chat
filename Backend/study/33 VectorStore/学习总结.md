# 向量数据库（VectorStore）知识总结

这个目录包含了4个Python文件，展示了LangChain中向量数据库的核心检索功能。

## 1. 带阀值的相似性搜索

### 知识点
- **FAISS向量数据库**：Facebook AI Similarity Search，高效的向量相似性搜索库
- **相似性得分阈值**：只返回相似度超过指定阈值的结果
- **similarity_search_with_relevance_scores()**：返回文档及其相关性得分

### 用途
当需要过滤掉低相似度结果时使用，确保返回的结果与查询足够相关。

### 示例代码
```python
from langchain_community.vectorstores import FAISS
from langchain_core.documents import Document
from langchain_community.embeddings import QianfanEmbeddingsEndpoint

# 创建文档集合
documents = [
    Document(page_content="笨笨是一只很喜欢睡觉的猫咪", metadata={"page": 1}),
    Document(page_content="我喜欢在夜晚听音乐，这让我感到放松。", metadata={"page": 2}),
    # ... 更多文档
]

# 构建向量数据库
db = FAISS.from_documents(documents, embedding)

# 带阈值的相似性搜索（只返回相似度>0.4的结果）
results = db.similarity_search_with_relevance_scores(
    "我养了一只猫，叫笨笨",
    score_threshold=0.4
)
```

## 2. as_retriever检索器

### 知识点
- **as_retriever()**：将向量数据库转换为标准检索器接口
- **search_type参数**：
  - `similarity`：标准相似性搜索
  - `similarity_score_threshold`：带阈值的相似性搜索
  - `mmr`：最大边际相关性搜索
- **search_kwargs参数**：
  - `k`：返回结果数量
  - `score_threshold`：相似度阈值
- **ConfigurableField**：使检索器配置可在运行时动态修改

### 用途
在RAG（检索增强生成）应用中，向量数据库通常需要转换为检索器才能与LLM链集成。

### 示例代码
```python
from langchain_weaviate import WeaviateVectorStore
from langchain_core.runnables import ConfigurableField

# 将向量数据库转换为检索器
retriever = db.as_retriever(
    search_type="similarity_score_threshold",
    search_kwargs={"k": 10, "score_threshold": 0.5},
).configurable_fields(
    search_type=ConfigurableField(id="db_search_type"),
    search_kwargs=ConfigurableField(id="db_search_kwargs"),
)

# 使用默认配置检索
documents = retriever.invoke("关于配置接口的信息有哪些")

# 运行时动态修改为MMR搜索
mmr_results = retriever.with_config(
    configurable={
        "db_search_type": "mmr",
        "db_search_kwargs": {"k": 4}
    }
).invoke("查询内容")
```

## 3. 最大边际相关性搜索（MMR）

### 知识点
- **最大边际相关性（MMR）**：平衡结果的相关性和多样性
- **max_marginal_relevance_search()**：执行MMR搜索
- **文档加载与分割**：UnstructuredMarkdownLoader + RecursiveCharacterTextSplitter
- **WeaviateVectorStore**：云端向量数据库服务

### 用途
当需要避免返回过于相似的结果时使用，例如问答系统需要提供多样化的答案。

### 示例代码
```python
from langchain_weaviate import WeaviateVectorStore
from langchain_community.document_loaders import UnstructuredMarkdownLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter

# 1. 加载并分割文档
loader = UnstructuredMarkdownLoader(file_path)
text_splitter = RecursiveCharacterTextSplitter(
    separators=["\n\n", "\n", "。|！|？", ...],
    chunk_size=500,
    chunk_overlap=50,
)
chunks = text_splitter.split_documents(loader.load())

# 2. 存储到向量数据库
db = WeaviateVectorStore(client=..., index_name="DatasetDemo", ...)
db.add_documents(chunks)

# 3. 执行MMR搜索（结果更多样化）
search_documents = db.max_marginal_relevance_search("关于应用配置的接口有哪些？")
```

## 4. 向量数据库检索

### 知识点
- **运行时可配置检索器**：通过configurable_fields实现动态配置
- **with_config()方法**：在调用时修改检索器行为
- **检索器集成模式**：标准化的检索接口，便于与LangChain链集成

### 用途
需要根据不同场景动态调整检索策略的应用，例如：
- 不同的查询使用不同的k值
- 有时用相似性搜索，有时用MMR
- 根据用户偏好调整阈值

### 示例代码
```python
from langchain_core.runnables import ConfigurableField

# 创建可配置的检索器
retriever = db.as_retriever(
    search_type="similarity_score_threshold",
    search_kwargs={"k": 10, "score_threshold": 0.5},
).configurable_fields(
    search_type=ConfigurableField(id="db_search_type"),
    search_kwargs=ConfigurableField(id="db_search_kwargs"),
)

# 场景1：高精度检索（高阈值）
high_precision = retriever.with_config(
    configurable={
        "db_search_kwargs": {"k": 5, "score_threshold": 0.8}
    }
).invoke("查询")

# 场景2：多样化检索（MMR）
diverse_results = retriever.with_config(
    configurable={
        "db_search_type": "mmr",
        "db_search_kwargs": {"k": 10}
    }
).invoke("查询")
```

## 核心概念对比

| 特性 | 相似性搜索 | 阈值搜索 | MMR搜索 |
|------|-----------|---------|---------|
| 方法名 | `similarity_search` | `similarity_search_with_relevance_scores` | `max_marginal_relevance_search` |
| 关注点 | 最相关的前k个 | 相关度超过阈值的结果 | 相关性+多样性 |
| 结果数量 | 固定k个 | 变化（取决于阈值） | 固定k个 |
| 适用场景 | 一般检索 | 高质量要求 | 需要多样化结果 |

## 技术栈总结

### 向量数据库
- **FAISS**：本地向量数据库，适合小规模数据和快速原型开发
- **Weaviate**：云端向量数据库，适合生产环境和大规模数据

### 嵌入模型
- **千帆Embeddings**：百度千帆平台提供的文本嵌入服务

### 文档处理
- **UnstructuredMarkdownLoader**：加载Markdown格式文档
- **RecursiveCharacterTextSplitter**：递归文本分割器，支持多种分隔符

### 检索策略
1. **相似性搜索**：基于向量相似度返回最相关的结果
2. **阈值搜索**：过滤低相似度结果，保证质量
3. **MMR搜索**：在相关性和多样性之间取得平衡

## 实际应用场景

### RAG应用开发流程
1. **文档准备**：加载文档 -> 文本分割 -> 向量化
2. **向量存储**：选择合适的向量数据库（FAISS/Weaviate）
3. **检索器配置**：转换为检索器，配置检索策略
4. **动态调整**：根据场景需求动态调整检索参数
5. **结果生成**：将检索结果输入LLM生成答案

### 参数选择建议
- **k值选择**：一般取3-10个结果，太多会引入噪声
- **阈值设置**：0.5-0.8之间，根据数据质量调整
- **chunk_size**：300-1000字符，根据文档类型调整
- **chunk_overlap**：一般为chunk_size的10-20%
